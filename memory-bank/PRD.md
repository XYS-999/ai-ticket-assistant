## 头脑风暴（发散，不等于都做）
目标用户：
普通员工（提单/查看进度/补充信息）
IT 支持（受理/处理/流转/回复）
管理员/审计人员（权限管理/审计查询）

典型场景：
员工提交“系统不可用/账号权限申请/软件安装”工单→系统记录审计→分配处理人→处理人流转状态并回复→员工确认→关闭
管理员配置角色与权限→所有关键操作可追溯（谁在何时做了什么）

功能候选池（先发散）：
认证鉴权：登录、JWT、刷新/退出（占位）、接口鉴权
权限体系：RBAC（用户-角色-权限），接口级权限校验
工单闭环：创建/查询/评论/指派/状态流转（先简化）
合规审计：关键操作审计日志（登录、创建/流转工单、权限变更）
工程化：Docker Compose 一键启动、OpenAPI 文档

MVP（第1个月/4周必须做，最小集合）：
M1：JWT 登录 + 接口鉴权（Spring Security 落地）
M2：RBAC 权限模型 + 权限校验（管理员配置、接口受控）
M3：审计日志（关键操作留痕，可查询）+ 工单最小闭环（可演示）

不做清单（明确不做，防止失控）：
N1：复杂前端/大屏报表（先用 Swagger + 极简接口演示）
N2：微服务拆分/分布式事务（先单体、模块化设计预留扩展点）

- 工程基建里程碑（已完成，PR-001）：后端 Spring Boot 最小骨架 + 健康检查 /api/health + CI 自动跑测试（细节见 TechDesign / QualityGate）
